# 文档验收清单

通过文件读取、命令执行、链接检查等方式验证文档质量。

## A. 完整性

| # | 检查项 | 操作 | FAIL 条件 |
|---|--------|------|----------|
| A1 | 文件存在 | 确认所有声明的文档文件路径存在 | 文件缺失、路径错误 |
| A2 | 章节完整 | 对照 Issue 要求的章节列表检查 | 缺少必要章节、章节为空占位（TODO / TBD） |
| A3 | 目录一致 | 如有目录/TOC，对比实际章节 | 目录项指向不存在的章节、实际章节未在目录中列出 |
| A4 | 内容充实 | 检查每个章节的内容量 | 章节只有标题无内容、内容少于 3 行（过于空洞） |

## B. 准确性

| # | 检查项 | 操作 | FAIL 条件 |
|---|--------|------|----------|
| B1 | 代码示例可运行 | 提取 code block，实际执行 | 语法错误、依赖缺失、运行报错 |
| B2 | 命令可执行 | 提取 bash/shell 命令，实际执行 | 命令不存在、参数错误、权限不足 |
| B3 | 文件路径正确 | 检查文档中引用的文件路径 | 路径指向不存在的文件 |
| B4 | 数据/数字准确 | 对照实际代码/配置核实文档中的数字 | 版本号过时、配置值与实际不一致 |
| B5 | 技术描述准确 | 核心技术概念是否与实际实现一致 | 架构描述与代码不符、API 签名与实际不匹配 |

## C. 链接有效性

| # | 检查项 | 操作 | FAIL 条件 |
|---|--------|------|----------|
| C1 | 内部链接 | 检查 `[text](relative/path)` 格式的链接 | 链接指向不存在的文件或锚点 |
| C2 | 外部链接 | 检查 `https://` 开头的链接（抽查，非全量） | 404、域名不可达 |
| C3 | 图片引用 | 检查 `![alt](path)` 格式的图片 | 图片文件不存在、路径错误 |

## D. 清晰度与质量

| # | 检查项 | 操作 | FAIL 条件 |
|---|--------|------|----------|
| D1 | 术语一致 | 扫描全文关键术语 | 同一概念使用不同名称（如混用 "harness" 和 "评测框架"） |
| D2 | 无歧义 | 检查关键步骤和决策描述 | 步骤描述模糊（"适当配置"而不说具体值）、条件判断不明确 |
| D3 | 可跟随性 | 按步骤文档从头操作 | 步骤之间缺少衔接、跳跃过大（假设读者已知未说明的前置条件） |
| D4 | 语法与排版 | 检查 Markdown 渲染 | 标题层级混乱、列表格式错误、代码块未标注语言、Markdown 语法错误导致渲染异常 |

## E. 格式质量

| # | 检查项 | 操作 | FAIL 条件 |
|---|--------|------|----------|
| E1 | Markdown 渲染 | 用 Playwright 打开 GitHub preview 或本地渲染 | 表格渲染错误、代码块溢出、图片不显示 |
| E2 | 代码块语言标注 | 检查所有 ``` 代码块 | 未标注语言（无语法高亮）、标注语言与内容不匹配 |
| E3 | 中英文排版 | 检查混排规范 | 中英文之间无空格、全角半角标点混用 |
