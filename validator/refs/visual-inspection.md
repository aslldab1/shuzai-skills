# 视觉检查协议

> **核心原则：截图是输入，不是证据。** 每次截图后必须停下来分析它。不分析截图等于没看。

## 1. 全页视觉扫描（每个页面必做）

导航到页面后，**不要点任何东西**，先完成以下分析：

### 1.1 截全页图并回答结构化问题

```
browser_take_screenshot(fullPage=true)
```

截图后必须逐条回答（不可跳过）：

| # | 问题 | FAIL 条件 |
|---|------|----------|
| V1 | 页面是否需要水平滚动才能看到完整内容？ | 任何内容超出 viewport 宽度 |
| V2 | 页面整体视觉层次是否清晰？（标题→内容→辅助信息） | 没有明确的视觉重心，信息平铺无层次 |
| V3 | 各区块之间的间距是否均匀、节奏一致？ | 区块间距差异 >2 倍（如 A-B 间 40px，B-C 间 8px） |
| V4 | 是否有区块视觉比例失调？（占据大面积但内容很少，或内容密集到挤压） | 某区块占页面 >40% 面积但只有 1-2 行内容 |
| V5 | 卡片/列表项等重复元素的尺寸是否一致？ | 同类卡片大小明显不同、某个卡片异常高/窄 |
| V6 | 文字是否清晰可读？（字号、对比度、行距） | 正文 <12px、浅灰文字在白底上、行距过密 |
| V7 | 图表/数据可视化是否完整渲染？（非空白、非静态占位） | 图表区域空白、只有坐标轴没有数据、图例与数据不匹配 |

### 1.2 逐区块放大检查

将页面按视觉区块拆分（header、hero、card group、chart section、footer 等），对每个区块：

```
browser_take_screenshot(element="区块描述", ref="区块ref")
```

检查：
- 内部元素是否对齐（左边缘齐、垂直间距均匀）
- 文字是否截断、溢出、重叠
- 图标/图片是否正确加载（无 broken image、无占位方框）
- 元素间距是否合理（不拥挤、不空旷）
- 配色是否与页面整体一致

**记录格式：**
```
[PASS|FAIL|WARN] V-{页面简称}-{区块名}
观察: {从截图中实际看到的情况}
问题: {具体哪里不对}（FAIL/WARN 时填写）
```

## 2. 视口适配检查

### 默认视口：1280×800（必测）

```
browser_resize(width=1280, height=800)
browser_navigate(url)
browser_take_screenshot(fullPage=true)
```

检查：
- 内容是否被压缩到不可读
- 卡片是否被挤成窄条
- 水平滚动条是否出现
- 导航是否被截断

### 扩展视口（按需）

仅当 Issue 声明支持响应式、或产品面向终端用户时测试：

| 视口 | 尺寸 | 何时测 |
|------|------|--------|
| 宽屏 | 1920×1080 | 设计稿为宽屏设计时 |
| 窄屏 | 1024×768 | 用户可能用小屏笔记本时 |
| 平板 | 768×1024 | Issue 声明响应式时 |

## 3. 设计稿对比（当 Stitch/HTML 设计稿存在时）

### 定位设计稿

从 Issue 正文中查找：
- `data/stitch-designs/` 路径下的 HTML 文件
- Stitch Project URL
- 设计规格文档路径

### 对比方法

1. 在 Playwright 中打开设计稿 HTML：
   ```
   browser_navigate(file:///path/to/design.html)
   browser_take_screenshot(fullPage=true, filename="design-{page}.png")
   ```

2. 在 Playwright 中打开实际页面：
   ```
   browser_navigate(http://localhost:xxxx/{page})
   browser_take_screenshot(fullPage=true, filename="actual-{page}.png")
   ```

3. 逐项对比两张截图：

| 维度 | 对比方式 | FAIL 阈值 |
|------|---------|----------|
| 布局结构 | 区块排列顺序和比例 | 缺少设计稿中的区块、顺序完全不同 |
| 配色方案 | 主色、背景色、强调色 | 主色明显偏离（非同色系） |
| 字体层级 | 标题/正文/辅助文字的大小关系 | 层级反转（小标题比大标题更大） |
| 组件风格 | 按钮、卡片、表格的样式 | 设计稿圆角卡片但实现为直角边框 |
| 信息密度 | 同一区域的内容量 | 设计稿留白充足但实现过于拥挤（或反之） |

**容忍度：**
- WARN：细微色差、字体替换（Manrope→Inter 等系统字体 fallback）、间距 ±20%
- FAIL：缺失区块、布局结构不同、配色体系不同、组件风格冲突
