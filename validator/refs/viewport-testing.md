# 视口适配测试

## 测试矩阵

| 视口 | 尺寸 | 何时测 | 说明 |
|------|------|--------|------|
| **桌面标准** | 1280×800 | **必测** | 大多数 Web 应用的目标宽度 |
| 桌面宽屏 | 1920×1080 | 设计稿为宽屏、或首次测试发现内容过窄时 | 检查内容是否合理拉伸/居中 |
| 窄屏笔记本 | 1024×768 | Dashboard/数据密集型应用 | 检查卡片/表格是否被压缩 |

> 移动端/平板端仅当 Issue 明确声明"响应式"时测试。Streamlit 应用默认不要求移动端适配。

## 每个视口的检查项

### 必检项（所有视口）

```
browser_resize(width={W}, height={H})
browser_navigate(url)
browser_take_screenshot(fullPage=true, filename="viewport-{W}-{page}.png")
```

| # | 检查项 | FAIL 条件 |
|---|--------|----------|
| VP-1 | 水平滚动 | 页面出现水平滚动条（content 宽度 > viewport 宽度） |
| VP-2 | 内容可读性 | 文字因挤压变得不可读、表格列重叠 |
| VP-3 | 卡片/区块完整性 | 卡片被压缩成窄条（宽度 <200px）、内容溢出卡片边界 |
| VP-4 | 图表可用性 | 图表坐标轴标签被截断、数据点重叠到无法区分 |
| VP-5 | 导航可用性 | 导航项被截断、挤出可见区域 |

### 宽屏特有检查

| # | 检查项 | FAIL 条件 |
|---|--------|----------|
| VP-W1 | 内容居中 | 内容贴左对齐而非居中，右侧大片空白 |
| VP-W2 | 最大宽度 | 内容无限拉宽导致行文过长（>120 字符/行） |

## 执行流程

1. 先在 1280×800 完成全部验收（Phase A/B/C）
2. 验收完成后，切换到其他需要的视口
3. 在每个额外视口只做 Phase A（视觉扫描）— 不重复交互测试
4. 记录视口特有的 FAIL/WARN 项
