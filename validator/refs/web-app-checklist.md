# Web App 验收清单

使用 Playwright 浏览器自动化逐项验证。每项必须有截图或 console 输出作为证据。

> **执行顺序：先 Phase A（视觉扫描），再 Phase B（用户旅程），最后 Phase C（系统性测试）。**
> Phase C 只需覆盖 Phase A/B 未触及的项。

## A. 可访问性（阻断级 — 任一失败则整体 FAIL）

| # | 检查项 | 操作 | FAIL 条件 |
|---|--------|------|----------|
| A1 | 首页加载 | `navigate` → 首页 URL，等待加载完成 | HTTP 非 200、白屏、加载超时 >10s |
| A2 | 所有页面可达 | 逐个 `navigate` 到每个路由/页面 | 任何页面返回 404/500/白屏 |
| A3 | Console 无 error | 每个页面 `console_messages` 过滤 error 级别 | 存在任何 console error（Streamlit 内部 _stcore 路径除外） |
| A4 | 无未捕获异常 | 检查页面是否有 error boundary / crash 信息 | 页面显示 stack trace 或"Something went wrong" |

## B. 视觉质量（Phase A 执行）

> → 完整方法论见 refs/visual-inspection.md

### B-I. 全页视觉分析（每个页面必做）

对每个页面截全页图后，逐条回答：

| # | 检查项 | FAIL 条件 |
|---|--------|----------|
| B1 | 水平溢出 | 内容超出 viewport 宽度，出现水平滚动条 |
| B2 | 视觉层次 | 无明确的标题→内容→辅助信息层级，信息平铺 |
| B3 | 间距节奏 | 区块间距差异 >2 倍，节奏不一致 |
| B4 | 区块比例 | 某区块占 >40% 面积但内容稀少，或内容挤压 |
| B5 | 重复元素一致性 | 同类卡片/列表项尺寸明显不一致 |
| B6 | 文字可读性 | 正文 <12px、对比度不足、行距过密 |
| B7 | 图表完整性 | 图表区域空白、只有坐标轴、数据与图例不匹配 |

### B-II. 逐区块检查

对页面中的每个视觉区块（header、card group、chart section、table、footer 等）：

| # | 检查项 | FAIL 条件 |
|---|--------|----------|
| B8 | 内部对齐 | 元素左边缘不齐、垂直间距不均 |
| B9 | 内容溢出 | 文字/数据超出容器边界，被截断或重叠 |
| B10 | 图标/图片 | 图片 broken、图标缺失显示方框 |
| B11 | 配色一致 | 区块配色与页面整体不一致，出现突兀颜色 |

### B-III. 视口适配

> → 完整方法论见 refs/viewport-testing.md

| # | 检查项 | FAIL 条件 |
|---|--------|----------|
| B12 | 1280px 布局 | 桌面标准视口下内容被压缩或溢出 |
| B13 | 卡片/表格完整 | 卡片宽度 <200px、表格列重叠 |

### B-IV. 设计稿对比（当存在 Stitch/HTML 设计稿时）

> → 完整方法论见 refs/visual-inspection.md 第 3 节

| # | 检查项 | FAIL 条件 |
|---|--------|----------|
| B14 | 布局结构匹配 | 缺少设计稿中的区块、区块顺序不同 |
| B15 | 配色方案匹配 | 主色明显偏离设计稿（非同色系） |
| B16 | 组件风格匹配 | 设计稿圆角但实现直角、阴影风格不同 |

## C. 用户旅程 + 首印象（Phase B 执行）

> → 完整方法论见 refs/user-journey.md

### C-I. 首印象测试（每个页面必做）

| # | 检查项 | FAIL 条件 |
|---|--------|----------|
| C1 | 页面用途理解 | 5 秒内无法理解页面做什么 |
| C2 | 视觉重心合理 | 最显眼元素不是最重要内容 |
| C3 | 行动入口明确 | 无明确 CTA，不知道该从哪开始 |
| C4 | 首屏价值 | 首屏全是装饰，核心内容需要滚动 |
| C5 | 跨页面风格一致 | 不同页面视觉风格断裂 |

### C-II. 用户旅程端到端

| # | 检查项 | FAIL 条件 |
|---|--------|----------|
| C6 | 旅程可完成 | 用户无法从入口走到目标终态 |
| C7 | 操作入口可发现 | 关键操作需要滚动/搜索才能找到 |
| C8 | 操作反馈及时 | 操作后 >2 秒无任何反馈 |
| C9 | 结果符合预期 | 操作结果与用户期望不一致 |
| C10 | 位置感清晰 | 操作后不知道自己在哪（无面包屑/高亮） |

### C-III. 信息架构

| # | 检查项 | FAIL 条件 |
|---|--------|----------|
| C11 | 导航可用 | 导航项不可点击、被遮挡、无当前页高亮 |
| C12 | 页面编排合理 | 关键功能入口隐藏 >2 层导航 |
| C13 | 操作流程高效 | 简单任务需要 >5 步 |
| C14 | 信息分组合理 | 相关数据散布在不同位置 |
| C15 | 跨页面数据一致 | Dashboard 汇总数与 Results 明细数矛盾 |

## D. 系统性交互测试（Phase C 执行 — 补充覆盖）

> **只测试 Phase A/B 中未触及的元素。**

### D-I. 交互元素逐个测试

页面加载后，用 `browser_snapshot` 获取元素列表，对 Phase B 旅程中**未操作过的**可交互元素执行：

| 元素类型 | 必须执行的操作 | PASS 标准 | FAIL 条件 |
|---------|--------------|----------|----------|
| 按钮（button） | `browser_click` 点击 | 有明确反馈 | 点击后无任何变化 |
| 链接（link） | `browser_click` 跟随 | 到达正确目标 | 404、无响应 |
| 下拉框（select） | 切换**至少 2 个**选项 | 内容联动变化 | 切换后内容不变 |
| 单选/多选（radio/checkbox） | 点击**每个**选项 | 关联内容随选项变化 | 切换后页面无反应 |
| 输入框（input） | 输入测试值并触发 | 输入被接受 | 输入被忽略 |
| 展开/折叠（expander） | 点击展开 | 显示有意义的内容 | 内容为空 |
| 标签页/Tab | 点击每个 Tab | 内容面板切换 | 内容不变 |
| 图表（Plotly 等） | 悬停/点击数据点 | 显示 tooltip | 无交互响应 |

### D-II. 数据正确性

| # | 检查项 | FAIL 条件 |
|---|--------|----------|
| D1 | 数据非占位 | Lorem ipsum、全 0、NaN、undefined、null |
| D2 | 数据合理性 | 百分比 >100%、日期格式错误、分数超范围 |
| D3 | 数据一致性 | 跨页面同一数据展示不一致 |
| D4 | 图表数据匹配 | 图表数据与旁边文字/表格数值不一致 |

### D-III. Issue 特定检查

- 将 Issue 验收标准的每一条映射为一个验证项
- Phase A/B 中已覆盖的标记为已验证
- 未覆盖的逐条独立验证

## E. 错误处理与边界（Phase C 补充）

| # | 检查项 | 操作 | FAIL 条件 |
|---|--------|------|----------|
| E1 | 空状态 | 在无数据条件下访问（如可能） | 白屏、报错（应显示友好提示） |
| E2 | 快速重复操作 | 连续多次点击按钮 | 重复提交、页面崩溃 |
